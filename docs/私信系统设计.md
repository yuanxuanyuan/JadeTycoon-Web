# 私信系统设计文档

## 一、概述

私信系统模拟微信聊天，用于 NPC 主动联络玩家，玩家通过回复选项推进对话、影响好感度或触发资金变动。私信与 **NPC 人脉**、**直播热度** 关联。

---

## 二、功能点

### 2.1 消息来源（触发方式）

| 触发条件 | 概率/条件 | 说明 |
|---------|----------|------|
| **每日随机私信** | 25% | 每进入新的一天时，`Math.random() < PHONE_MSG_CHANCE` 随机选一条 NPC 私信入队 |
| **榜一大哥私聊** | 100%（条件满足时） | 直播热度连续 3 天满值（100）时触发，**全局仅触发一次** |

### 2.2 消息类型

| 类型 | 来源 | 内容 | 选项效果 |
|-----|------|------|---------|
| **borrow** 借钱 | 老王 lao_wang | 借 5000 周转 | 同意：好感 +2、扣款 5000、进入下一句；拒绝：好感 -1 |
| **gossip** 八卦 | 小美 xiao_mei | 矿难涨价囤货 | 早囤了：好感 +1；没囤：好感不变 |
| **order** 订货 | 陈师傅 master_chen | 冰种观音溢价 15% | 有货：好感 +3；没有：好感不变 |
| **gossip** 赞助 | 网红博主 influencer | 切石专场赞助公斤料 | 送两块：好感 +2；下次：好感 -1 |
| **榜一大哥** | bang_yige | 看中好料想收 | 改天聊：无效果，仅关闭对话 |

### 2.3 对话树结构

- 支持多轮对话：`options[].nextId` 指向下一则消息模板
- 仅 `borrow_1` → `borrow_1_ok` 有 nextId 链
- 其余消息为单轮，选完即结束

## 三、关联性

### 3.1 与 NPC 人脉的关系

```
玩家选择回复选项
    → opt.affinityDelta 生效
    → 更新 npcRelations[npcId]
    → 影响：NPC 出价、技能解锁、好感等级
```

- **好感变化**：`affinityDelta` 直接加到 `npcRelations[npcId]`
- **仅对人脉系统中的 NPC 生效**：`bang_yige` 不在 `npcRelations` 中，其选项的 `affinityDelta` 不生效

### 3.2 与直播热度的关系

```
直播热度 ≥ 100
    → livestreamHypeFullDays 累加
    → 连续 3 天满值
    → 触发榜一大哥私信（bangYigeTriggeredRef 标记，不再触发）
```

- 依赖 `livestreamHype`、`livestreamHypeFullDays`、`LIVESTREAM_CUT.hypeFullDaysForBangYige`

### 3.3 与资金的关系

| 场景 | 条件 | 效果 |
|-----|------|------|
| 老王借钱 | `msgId === 'borrow_1'` 且选择「没问题，拿去用」 | 扣款 5000 |

### 3.4 与存档的关系

- `phoneMessages` 参与存档/读档
- `bangYigeTriggeredRef` 不存档，新游戏时重置

## 四、数据结构

### 4.1 单条消息

```ts
{
  id: number;        // 唯一标识，用于排序、查找
  npcId: string;     // 发信人 id（lao_wang / xiao_mei / master_chen / influencer / bang_yige）
  msgId?: string;    // 模板 id，用于业务逻辑（如 borrow_1 判断扣款）
  msg: string;       // 展示文案
  isPlayer?: boolean;// 是否玩家发出的
  reply?: string;    // 玩家选中的回复文案
  options?: Array<{
    text: string;
    affinityDelta: number;
    nextId: string | null;
    reply: string;
  }>;
}
```

### 4.2 消息模板（PHONE_MSG_POOL / BANG_YIGE_MSG）

```ts
{
  id: string;
  npcId: string;
  type: 'borrow' | 'gossip' | 'order';
  msg: string;
  options: [...];
}
```

## 五、UI 与交互

### 5.1 入口

- 底部栏 **📱 私信** 按钮
- 有待回复会话时显示红点数字（待回复会话数）

### 5.2 私信面板（PhonePanel）

- 仿微信样式，聊天气泡
- 按 NPC 分会话，支持多会话切换
- 会话按**最近一条消息时间**排序
- 最后一条为 NPC 且有选项时，显示回复按钮

### 5.3 红点逻辑

- 统计：每个 NPC 最后一条消息中，**来自 NPC 且仍有 options** 的会话数
- 全回复后红点消失

## 六、常量配置

| 常量 | 值 | 说明 |
|-----|---|------|
| PHONE_MSG_CHANCE | 0.25 | 每日随机私信触发概率 |
| PHONE_MSG_STARTERS | ['borrow_1','gossip_1','order_1','gossip_2'] | 随机池 |
| BORROW_AMOUNT | 5000 | 老王借钱金额（逻辑内联） |
| hypeFullDaysForBangYige | 3 | 直播热度满值天数阈值 |
